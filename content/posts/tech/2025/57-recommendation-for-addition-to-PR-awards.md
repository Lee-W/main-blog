👉 原文: [Recommendation for addition to PR awards](https://lists.apache.org/thread/rz28lgots24pq54x67stdyyrq6848ptm)

## 本文
既然已經有本月 PR 了
那要不要再加一個不可重複得獎的新手 PR 獎 (Rookie PR)
獎項的提名可以透過在 PR 留言 `#rookieprotm`
或者由貢獻者自己提名

目前的結論依然希望每個月只有一個 PR 可以得獎
但可以分不同的領域 (e.g., UI, providers)
至於領域要怎麼分，倒是還沒有決定

何謂新手則是尚未明確定義，討論中有提到

1. 第一個 PR
2. 前 N 個 PR
3. 不在[百人榜](https://github.com/apache/airflow/graphs/contributors)上的人都可以（目前是 35 個提交）
4. 至少 N 個 PR 以上

發起者 Srabasti 願意負責當這個獎項的執行者
並在今早發出了第一封提名邀請 [November 2025 Rookie PR of the Month - NEW AWARD ADDITION](https://lists.apache.org/thread/hswcpkkljwbn4984t9x73xw2xhghb5m3)
如果之後新手 PR 獎由像是 Srabasti 這樣的社群新夥伴執行
可能也能讓貢獻者提名自己的心理門檻降低

## 我怎麼想
何謂新手中的 2 跟 3 就是我提的 😆
結合不在[百人榜]跟 N 個 PR 以上是我心中最好的做法
N 可以是 5 或 10 都好

每次衝刺開發或工作坊結束後最困難的就是讓人留下來
也許這能激勵新人再多堅持一陣子

另外一點，比較需要花點心力的是宣傳「提名自己」這件事
初期大概還是需要一些人推個幾把，讓這個風氣起來
大概就跟 PyCon TW 需要一直推廣大家可以申請財務補助那樣吧
雖然現在 PyCon TW 也沒錢錢補助太多人惹嗚嗚嗚 😢

[百人榜]: https://github.com/apache/airflow/graphs/contributors
