Wei Lee

Following up on Achieve 300 contributions in Apache Airflow.
Just a quick reflection on what I did between 300 and 400 contributions.

The count is now quite inaccurate. I never really understood the difference between the PR I created and the number of commits I contributed. However, I decided to include PRs from feat(api_fastapi): include asset ID in asset nodes when calling "/ui/dependencies" and "/ui/structure/structure_data" #47381 to build(pre-commit): upgrade node version to 22.18.0 and flynt 1.0.6 #54009 as part of my 300 to 400 contributions. This should be approximately correct. Next time, we'll start from build(pre-commit): upgrade node version to 22.18.0 and flynt 1.0.6 #54009.

This time, I created the following script to help me list the PRs I need in my preferred format. Itâ€™s quite convenient.

START_PR_NUMBER=47381
END_PR_NUMBER=54009

START_MERGED_AT=$(gh pr view $START_PR_NUMBER --json mergedAt --jq .mergedAt)
END_MERGED_AT=$(gh pr view $END_PR_NUMBER --json mergedAt --jq .mergedAt)

echo "Start merged at: $START_MERGED_AT"
echo "End merged at:   $END_MERGED_AT"

gh pr list \
        -A @me \
        -s merged \
        --json title,mergedAt,number,url \
        -L 1000 \
        --jq "[.[]
                | select(.mergedAt > \"$START_MERGED_AT\" and .mergedAt < \"$END_MERGED_AT\")]
                | sort_by(.mergedAt)[]
                | \"1. [\(.title) #\(.number)](\(.url))\"" |
        > output.md
